# Introduction to Systems of ODEs

## The Big Picture

Most real engineering problems involve *multiple* coupled quantities — two masses connected by a spring, three reacting chemical species, voltages and currents in a circuit network. Each quantity changes due to the others, leading to a **system of coupled ODEs**. Linear systems can be written as $\dot{\mathbf{x}} = A\mathbf{x}$ and solved using eigenvalues — turning calculus into linear algebra. This connects ODE theory to matrix analysis and reveals why eigenvalues control the qualitative behavior of dynamic systems.

## Standard Form

A linear system of $n$ first-order ODEs:

$$\frac{d\mathbf{x}}{dt} = A\mathbf{x} + \mathbf{f}(t), \quad \mathbf{x}(0) = \mathbf{x}_0$$

where $\mathbf{x}\in\mathbb{R}^n$ and $A$ is $n\times n$. Homogeneous case: $\mathbf{f} = 0$.

## Reduction of Higher-Order ODEs

Any $n$-th order linear ODE can be rewritten as a system of $n$ first-order ODEs.

**Example.** $y'' + 2y' + 5y = 0$. Let $x_1 = y$, $x_2 = y'$. Then:

$$\dot x_1 = x_2, \quad \dot x_2 = -5x_1 - 2x_2$$

In matrix form: $\dot{\mathbf{x}} = \begin{pmatrix} 0 & 1\\ -5 & -2\end{pmatrix}\mathbf{x}$.

This rewrite is fundamental for numerical methods (Runge-Kutta, etc.) which solve first-order systems.

## Phase Portraits

For 2D systems, plot trajectories in the $(x_1, x_2)$ plane. Different dynamics produce different geometric patterns:
- **Stable spiral:** trajectories spiral inward (damped oscillator)
- **Unstable spiral:** outward spiral (undamped, energy injection)
- **Stable node:** trajectories converge along axes
- **Saddle:** trajectories approach then diverge
- **Center:** closed orbits (undamped oscillator)

The *eigenvalues* of $A$ classify the equilibrium type.

## Equilibria and Linearization

For nonlinear $\dot{\mathbf{x}} = \mathbf{f}(\mathbf{x})$: equilibria are points where $\mathbf{f} = \mathbf{0}$.

Near an equilibrium, linearize: $\dot{\delta\mathbf{x}} \approx J\delta\mathbf{x}$, where $J$ is the Jacobian of $\mathbf{f}$. The eigenvalues of $J$ determine local stability.

This **linear stability analysis** is the workhorse of dynamical systems theory — used in fluid stability, control theory, ecology.

## Common Misconceptions

**Misconception 1: "Coupled ODEs need each variable solved separately."**

Coupled systems must be treated as a *vector*. Trying to solve one variable at a time fails because each depends on the others. The matrix approach handles them simultaneously.

**Misconception 2: "Higher-order ODEs are different from systems."**

They're equivalent — first-order systems can encode any higher-order ODE. Numerical libraries usually accept only first-order systems.

**Misconception 3: "Phase portraits are just pretty pictures."**

They visualize *all possible trajectories at once* — an enormous amount of information. They reveal stability, bifurcations, and qualitative changes that are hard to see from explicit solutions.

## Engineering Connection

- **Multi-DOF vibration:** $n$-mass spring system gives a 2nd-order system in $\mathbb{R}^n$, equivalent to a first-order system in $\mathbb{R}^{2n}$.
- **State-space representation in control:** $\dot{\mathbf{x}} = A\mathbf{x} + B\mathbf{u}$ is the standard form in modern control theory.
- **Population models (predator-prey):** $\dot N_1 = aN_1 - bN_1 N_2$, $\dot N_2 = -cN_2 + dN_1 N_2$ — coupled, nonlinear.
- **Chemical kinetics:** chains of reactions give systems of first-order ODEs.
- **Electrical networks:** node and mesh equations form linear systems.

## Practice Problems

<ExQ>
Convert $y'' + 4y' + 3y = 0$ into a first-order system. Write the matrix.
</ExQ>
<ExA>
$x_1 = y, x_2 = y'$:
$\dot x_1 = x_2$
$\dot x_2 = -3 x_1 - 4 x_2$

Matrix: $A = \begin{pmatrix}0 & 1\\ -3 & -4\end{pmatrix}$.
</ExA>

<ExQ>
A system $\dot{\mathbf{x}} = A\mathbf{x}$ with $A = \begin{pmatrix}-1 & 0\\ 0 & -2\end{pmatrix}$. Solve given $\mathbf{x}(0) = (3, 5)^T$.
</ExQ>
<ExA>
$A$ is diagonal, so the components decouple:
$x_1(t) = 3 e^{-t}$
$x_2(t) = 5 e^{-2t}$

Both decay exponentially; $x_2$ decays twice as fast. Stable node.
</ExA>

<ExQ>
**Engineering: Two-tank mixing.** Tank 1 holds $V_1 = 100$ L, Tank 2 holds $V_2 = 200$ L. Salt amounts $S_1, S_2$ (kg). Tank 1 receives pure water at 5 L/min and feeds Tank 2 at 5 L/min; Tank 2 drains at 5 L/min. Set up the ODE system.
</ExQ>
<ExA>
Tank 1: rate of salt change = (in) - (out) = 0 - (5/100)$S_1$ = $-S_1/20$.

Tank 2: rate = (5/100)$S_1$ - (5/200)$S_2$ = $S_1/20 - S_2/40$.

System:
$\dot S_1 = -S_1/20$
$\dot S_2 = S_1/20 - S_2/40$

Triangular system: solve $S_1$ first (exponential decay), then $S_2$ via integrating factor.
</ExA>

<ExQ>
**Conceptual.** Why do we use first-order systems for numerical integration even when the original equation is higher-order?
</ExQ>
<ExA>
Numerical methods like Runge-Kutta were developed for first-order systems. Their convergence theory and error analysis assume this form.

By rewriting an $n$-th order ODE as $n$ first-order equations, we can apply RK4, adaptive solvers, etc., directly. This is why all numerical libraries (MATLAB's ode45, SciPy's solve_ivp) take first-order systems as input.

The conversion costs nothing mathematically and makes the entire numerical apparatus available.
</ExA>
