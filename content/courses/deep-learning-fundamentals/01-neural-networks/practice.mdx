# Chapter 1 Practice — Neural Networks

<ExQ>
**1.** A neuron has weights $w = (1, 2, -1)$, bias $b = 0.5$, ReLU activation. Find output for $x = (2, 1, 3)$.
</ExQ>
<ExA>
$z = w\cdot x + b = 2 + 2 - 3 + 0.5 = 1.5$.

ReLU: $\max(0, 1.5) = 1.5$.
</ExA>

<ExQ>
**2.** Why is ReLU preferred over sigmoid in modern deep networks?
</ExQ>
<ExA>
- ReLU has gradient 1 for $z > 0$, avoiding vanishing gradient problem.
- Sigmoid saturates: gradient $\to 0$ for large $|z|$, blocking learning.
- ReLU is also faster to compute.

For deep networks, ReLU enables training of much deeper architectures than sigmoid.
</ExA>

<ExQ>
**3.** A 3-layer MLP: 5 inputs, 64 hidden, 32 hidden, 1 output. Total parameter count?
</ExQ>
<ExA>
Layer 1: $5\cdot 64 + 64 = 384$.
Layer 2: $64\cdot 32 + 32 = 2080$.
Layer 3: $32\cdot 1 + 1 = 33$.

Total: $2497$ parameters.
</ExA>

<ExQ>
**4. Conceptual.** Why do we need activation functions between layers?
</ExQ>
<ExA>
Without nonlinearity, stacked linear layers collapse to a single linear transform: $A B C x = Mx$ where $M = ABC$. Just one effective layer — no expressive power gain.

Nonlinear activation breaks this collapse, allowing the network to express nonlinear functions of arbitrary complexity (universal approximation theorem).
</ExA>

<ExQ>
**5. Engineering.** Predict stress in a beam from 5 design parameters. Why might you choose [5, 64, 64, 1] over [5, 1000, 1] architecture (same number of parameters)?
</ExQ>
<ExA>
Deeper networks compose simpler features hierarchically; wider single-layer networks must memorize all combinations directly. Empirically, deeper architectures generalize better with the same parameter budget.

Also: deep nets often train faster (each layer has less work) despite having more layers.

This insight (depth > width for same parameters) drove the deep learning revolution.
</ExA>
